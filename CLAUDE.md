# Claude Code 工程规则

本文档为 Claude Code 修改本仓库时的默认项目规则。

## 必须执行的记录规则

每次对工程文件进行实际修改后，必须同步更新 `report/report.md`。用户不需要在每次对话中重复说明“默认每次修改都记录到 report.md”。

`report/report.md` 的记录内容必须包括：

1. 本次内容修改完成时间。
2. 本次修改的文件列表和具体修改位置。
3. 本次修改后项目相对于上一版项目的优化点。
4. 本次修改后仍存在的不足点。
5. 本次修改未完成或需要继续优化的部分。

新增记录应放在历史记录顶部，保持时间倒序。

## 工程约束

1. 不要提交真实图片数据、生成的 CSV、可视化图片或模型文件，除非用户明确要求。
2. 优先保持现有工程结构，不做无关重构。
3. 修改 Python 代码后至少运行 `python -m compileall src tests`。
4. 如果 `pytest` 可用，运行 `python -m pytest -q`。
