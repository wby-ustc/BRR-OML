# Agent 工程规则

本文档为本仓库的 AI agent 默认工作规则。使用 Codex 或其他支持仓库级指令文件的 agent 修改本工程时，应优先遵循本文档。

## 修改记录要求

每次对工程文件进行实际修改后，必须同步更新 `report/report.md`，不需要用户在每次对话中重复说明。

记录要求：

1. 在 `report/report.md` 顶部追加新的修改记录，按时间倒序排列。
2. 记录本次内容修改完成时间，格式建议为 `YYYY-MM-DD HH:MM`。
3. 记录本次修改的文件列表，并说明具体代码位置，例如函数名、配置节、标题名或行号范围。
4. 记录本次修改相对于上一版的优化点。
5. 记录本次修改后仍存在的不足点。
6. 记录本次未完成或需要继续优化的部分。
7. 如果本次没有产生文件修改，不需要新增修改记录。

## 工程约束

1. 不要提交真实图片数据到 Git 仓库。
2. 不要提交生成的 CSV、图片结果、模型文件，除非用户明确要求。
3. 优先复用现有模块和 `config.yaml` 参数。
4. 修改 Python 代码后至少运行：

```bash
python -m compileall src tests
```

5. 如果当前环境安装了 `pytest`，继续运行：

```bash
python -m pytest -q
```
